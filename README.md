# 🏀 Maccabi Intelligence Agent (MIA) & Daily News App

## 📝 Project Overview 
MIA is a sophisticated, full-stack Multi-Agent AI System and an accompanying native Android application. Built using the CrewAI framework and Google Gemini 2.5 Flash on the backend, and modern Kotlin/Jetpack Compose on the frontend. 

The system acts as an autonomous sports journalist, fact-checker, and radio broadcaster. It researches, cross-validates, filters, and summarizes the latest news regarding Maccabi Tel Aviv Football and Basketball Clubs. The validated data is converted into a structured JSON feed and a high-quality audio podcast, which are then seamlessly distributed to a private Telegram Bot and pushed to a Firebase Cloud database to be consumed by the dedicated Android mobile application.

## 🎯 Primary Objective 
To automate the daily consumption of sports news by creating an end-to-end "research-to-action" pipeline. The project eliminates fragmented media consumption and fake news by running raw data through a heuristic validation engine, and delivers a clean, verified, and beautifully designed user experience directly to the user's smartphone.

## 🏗️ System Architecture (Monorepo)

This repository is structured as a Monorepo containing both the AI Backend and the Android Client.

### 1. 🧠 The Backend & AI Middleware (AWS EC2 / Python)
* **The LLM Brain:** Powered by Gemini 2.5 Flash for high-speed, large-context processing of Google Search results.
* **Maccabi Specialist (The Researcher):** Gathers "first-to-report" news using the Serper API from local whitelisted sites, utilizing semantic classification to bypass poorly categorized Israeli sports menus.
* **The "רעיון של שמש" Engine (The Validator / Gatekeeper):** Groups news by entity and calculates a **Trust Score (0-100%)** based on a heuristic credibility dictionary. Detects consensus (+10% boost) or contradictions (drops to 40%, tagged `[CONFLICTING]`). Triggers targeted international API searches only when necessary to conserve quotas.
* **Sports Newsletter Editor (The Writer):** Translates findings into professional Hebrew, naturally weaves in primary sources, and formats the final output into a strict **JSON array**. Items falling below the confidence threshold are automatically excluded to prevent hallucinations.

### 2. ☁️ The Cloud Layer (Firebase)
* **Cloud Firestore:** Acts as the NoSQL real-time database. The Python backend pushes the daily JSON array as individual documents (NewsItems), which the Android app listens to via Kotlin Flows.
* **Cloud Storage:** Stores the daily `.mp3` podcast generated by Microsoft's `edge-tts`, providing a public URL for the Android media player.

### 3. 📱 The Frontend (Android Native / Kotlin)
* **UI Framework:** 100% Jetpack Compose for a declarative, responsive, and modern user interface.
* **Architecture:** Built strictly on **Clean Architecture** principles (Domain, Data, Presentation layers) utilizing the **MVVM** (Model-View-ViewModel) design pattern.
* **Dependency Injection:** Managed via **Dagger Hilt** for scalable and decoupled component lifecycle management.
* **Audio Engine:** Integrated **ExoPlayer** bound to a global `ViewModel`, enabling a persistent "Bottom Mini-Player" that allows users to listen to the podcast while seamlessly navigating the news feed.

## 🛠️ Technical Solutions & Engineering Highlights

* **API Optimization & Query Batching:** Reduced global search API calls from $O(N)$ to $O(1)$ by programming the Validator agent to aggregate entities into a single boolean `OR` string (e.g., `("Player A" OR "Player B")`).
* **Data Parsing & Schema Enforcement:** Transitioned the LLM output from unstructured Markdown to a strict JSON schema, enabling reliable NoSQL database population and structured UI rendering (Titles, Trust Badges, Source Links).
* **The "Robot Reading URLs" Error (Regex Filtering):** Solved the issue of the TTS engine reading aloud complex URLs by implementing a Python `re` filter to create a clean text layer specifically for the audio generator.
* **Cloud Migration & True Automation:** The backend runs 100% autonomously on an AWS EC2 Ubuntu Instance via a Linux `cron` job scheduled daily.
* **State Management (Android):** Utilized `StateFlow` and `Sealed Classes` (UiState) to safely handle asynchronous network calls, loading states, and error handling in the UI without blocking the main thread.

## 🚀 Future Expansion & Roadmap

* **Offline First (Android):** Implement **Room Database** in the Android app's Data Layer to cache the daily news, allowing blazing-fast load times even without internet connectivity.
* **Push Notifications:** Integrate Firebase Cloud Messaging (FCM) to trigger mobile notifications when "Breaking News" (high-confidence score transfers/firings) is detected.
* **Advanced Telegram Interactions:** Upgrade the bot from a "broadcast-only" channel to an interactive agent (RAG) where the user can reply and ask follow-up questions.
